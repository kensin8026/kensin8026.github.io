# 새 포트폴리오 사이트 구축 계획

## 목표
- skillsmp 스타일의 다크·대비 강한 구조를 가진 단일 페이지 포트폴리오 제작
- 기능: 소개, 프로젝트, 기술, 경력, 연락처를 한 페이지에서 빠르게 탐색 가능하게 구성
- Jekyll/GitHub Pages 배포 안정성 유지

## 범위
1. 기존 콘텐츠/레이아웃 완전 리셋
2. 새 브랜드 아이덴티티 결정(폰트, 컬러, 스페이싱, 애니메이션 톤)
3. 핵심 섹션(헤더, Hero, About, Projects, Skills, Experience, Contact, Footer) 구성
4. 반응형 처리(모바일 우선)
5. 접근성/메타태그/퍼포먼스 최소 기준 반영

## 단계별 작업
### 1) 기초 구조 설계
- `index.html` 단일 파일 기반 랜딩 구성
- 상단 내비게이션 + 섹션 앵커 링크
- 첫 인상(1초 내 로딩)용 큰 타이틀과 짧은 CTA

### 2) 비주얼 언어 확정
- 배경: 어두운 그라디언트 + 미묘한 노이즈/광택
- 포인트 색상: 블루/퍼플 하이라이트 1개
- 카드: 유리형 패널, 미세한 테두리/그림자
- 폰트: 사용자 지정 고딕 계열 + 강조체 조합

### 3) 콘텐츠 기획
- About: 문제 해결 접근법 3줄
- 프로젝트: 3~5개 카드(문제/해결/성과)
- 스킬: 카테고리별 점수 or 사용 빈도 바
- 경력: 타임라인
- Contact: 이메일/깃허브/링크드인 버튼

### 4) 인터랙션
- 섹션 reveal 애니메이션
- 스킬 바 애니메이션
- 네비게이션 현재 섹션 하이라이트(옵션)

### 5) 검토 및 배포 전 점검
- 레이아웃 깨짐 체크(모바일)
- 텍스트 대비/가독성
- 링크 정상 동작
- GitHub Pages 빌드 확인

## 산출물
- 새로 작성된 `index.html`
- 필요 시 `assets/styles.css` 분리
- 배포 직전 최종 `README.md` 초안
